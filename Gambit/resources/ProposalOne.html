 1.0   Project Overview

1.1.               Abstract

Gambit is a two player strategy and tactics game that allows both players to strategize a plan of movements and actions which will both be executed all at once in real time. The player who makes better tactical decisions, effectively positions their troops and takes out the other team first wins!

There are lots of strategy games in the market which employ an alternating turn system for combat, such as chess. However, we would like to create a fresh take on an old genre, where players plan and execute their actions simultaneously, taking a page from Rock-Paper-Scissors. Gambit aims to create a unique gameplay experience, different from either of these games by combining the management of a group of units in chess with the simultaneous decision-making of Rock-Paper-Scissors. In addition to smart strategic play, a successful player must actively plan what his team needs to do in response to what the enemy might possibly act on. These mechanics are typically associated with the genre of turn-based tactical games.

The final product of the computer game will be a fun strategy and tactic game that is essentially entertainment. We hope to provide a unique gaming experience by fusing elements from different popular genres.

1.4.               Scope and Objective

We will be using some libraries to help us with implementation of UI/animations, and remote communication. We plan to use Slick2D and LWJGL to help with the UI/animations, and RMI to help with connecting players.

  1. The user can run the game (as a .exe or an executable .jar) which opens to the main menu where they will be able to Connect or exit.
  1. The user connects to the game server if connect is chosen.
  2. The Menu closes if exit is chosen.
  3. A game session will be created for the users(2) when both are connected.
  1. In the Planning Phase, the user manages their units.
  1. The user can select a unit to view its status.
  2. The user can give commands to a selected unit
  1. The user can command units to Advance and Turn
  1. The user can see a summary of their commands
  2. When the user is ready to submit their actions, the “Ready!” button will submit their actions to the server.
  1. In the Execute Phase, the user receives the results of each turn.
  1. The game animates the actions of both sides
  1. Unit movement and shooting will be visualized with animations
  1. The game updates the status of each unit
  2. The game will continue repeating the Planning and Executing Phase until win condition is recognized.
  1. A player will win when all units of the opposing team have been eliminated.
  1. User will be able to request a rematch or to quit.
  1. If rematch is chosen, a new game will be made.
  2. If the user chooses to quit, the game will execute.
  3. A user cannot rematch if the other person chooses to quit.
